Basic operations on vectors

Norm

The norm (or magnitude) of a vector x=(x1,x2,⋯,xn)T is defined by |x|=√x21+x22+⋯+x2n

Normalized vector

The normalized (or unit) vector of x=(x1,x2,⋯,xn)T is a vector in the same direction but with norm 1: x=x|x|=(x1|x|,x2|x|,⋯,xn|x|)T

Scalar product

The scalar product between 2 vectors a=(ax,ay,az)T and b=(bx,by,bz)T is defined by


  • a⋅b=|a||b|cos(θ)
OR
  • a⋅b=axbx+ayby+azbz

2D transformations

We look for the operators that scale/rotate/translate a point or a vector P=(xy) into another one P′=(x′y′)

Scaling


Scaling transformation
Multiply each coordinate by a scaling factor:
(x′y′)=(sxxsyy)

MATRIX FORM: P′=SP
(x′y′)=(sx00sy)(xy)

Rotation

Rotation transformation
Rotate each vector around the origin:
(x′y′)=(cos(θ)x−sin(θ)ysin(θ)x+cos(θ)y)

MATRIX FORM: P′=RθP
(x′y′)=(cos(θ)−sin(θ)sin(θ)cos(θ))(xy)

Translation

Translation transformation
Translation is a simple vectorial sum
(x′y′)=(x+txy+ty)

MATRIX FORM: P′=P+T
(x′y′)=(xy)+(txty)

Notation issue

  • Scaling = matrix multiplication
  • Rotation = matrix multiplication
  • Translation = vectors addition
We would like to find a unique and unified representation for all transformations, so that we could concatenate multiple transformations.
How can we transform the translation into a matrix multiplication?

Homogeneous coordinates

  • Powerfull geometric tool, used in many applications: image, robotic, vision, etc.
  • We add a third coordinate w to a 2D point, which becomes a vector: (xy)=(xy1)=(xyw)
  • 2 points (x1,y1,w1)T and (x2,y2,w2)T are equal if and only if x1/w1=x2/w2 and y1/w1=y2/w2
  • A point is projected at infinity when w=0
    • It allows to represent affine spaces (and differentiate points and vectors)
    • It allows to represent projection transformations

Combining transformations

  • Simply by multiplying matrices
  • Every 2D transformation can be expressed with matrices with homogeneous coordinates
  • Very general notation
  • Example for a translation followed by a rotation
    • M=RθT
  • Rotation around a point q:
    • Translate to the origin: Tq
    • Rotate around the origin: Rθ
    • Translate back to q: −Tq


Rotation of p around q 1 translate to the origin: Tqp 2 rotate around the origin: RθTqp 3 translate back to q: (−Tq)RθTqp

3D transformations

  • Same principle
  • We use a fourth coordinate w: (xyz)=(xyz1)=(xyzw)
  • 2 points (x1,y1,z1,w1)T and (x2,y2,z2,w2)T are equal if and only if x1/w1=x2/w2,  y1/w1=y2/w2 and z1/w1=z2/w2

3D scaling: P′=SP

S=(sx0000sy0000sz00001)


3D translation: P′=TP

T=(100tx010ty001tz0001)


3D rotation: P′=Rx/y/zP

  • Depends on an axis and an angle
    • Simple expression around x, y, and z axis
    • Other rotations are expressed as combinations of these simple rotations
  • For a direct representation, use quaternions
Rx=(10000cosθ−sinθ00sinθcosθ00001)
 Ry=(cosθ0sinθ00100−sinθ0cosθ00001)
 Rz=(cosθ−sinθ00sinθcosθ0000100001)

  • Every 3D transformation can be expressed as combinations of translations, scalings and rotations !
    • can be represented by a 4×4 matrix that uses homogeneous coordinates
  • Multiplication of matrices is not commutative !
  • The order of transformations is important: RT≠TR

Transformations on complex objects

  • A complex object is defined as a combination of smaller objects
  • We want a behavior so that B and C should follow A when A moves
  • Solution: use relative coordinates:
    • position of B according to A
    • position of C according to B

Considering that "draw A" is drawing a rectangle centered at the origin and aligned on the x-axis, and T is a half length translation matrix, we may concatenate transformations:
  • M: initial matrix
  • M1=MRαT
  • draw A
  • M2=M1TRβT
  • draw B
  • M3=M2TRγT
  • draw C1
Question: how can we come back to M2 to draw C2 and C3?
Solution: keep track on all the previous matrices using a stack:
  • M: initial matrix
  • pushmatrix()
  • M=MRαT
  • draw A
  • M=MTRβT
  • draw B
  • pushmatrix()
  • M=MTR1γT
  • draw C1
  • popmatrix()
  • pushmatrix()
  • M=MTR2γT
  • draw C2
  • popmatrix()
  • pushmatrix()
  • M=MTR3γT
  • draw C3
  • popmatrix()
  • popmatrix()

  • Hierachical representation of objects.
  • Reusable objects (scene graph)

The model matrix

This matrix is defined by any combinations of the matrices seen above for transforming object coordinates.

The View matrix

We want to define a virtual camera that comprises:
  • a camera position e
  • an x-axis: the right vector r
  • a y-axis: the up vector u
  • a z-axis: the view vector v
the whole scene will this be defined in this coordinate system:
V=(rxux−vx−exryuy−vy−eyrzuz−vz−ez0001)

The right, up and view vectors can be easily found using more intuitive parameters:
  • a camera position e
  • the position of a reference point at which the camera is looking at c
  • an up-vector u′

  • The view vector v is given by v=(c−e)/|c−e|
  • The right vector r is given by r=v×u′
  • The up vector vector u is given by u=r×v

The projection matrix
Orthographic projection
  • Object sizes do not change
  • parallel lines are kept parallel
 Perspective projection
  • Far objects are smaller
  • Similar to the human eye

More generaly, the orthographic matrix can be defined using a 6-tuple (left,right,bottom,top,near,far), which defines the clipping plane:
P=(2right−left00−right+leftright−left02top−bottom0−top+bottomtop−bottom00−2far−near−far+nearfar−near0001)

which defines a translation, followed by a scaling, to obtain a unit cube centered at the origin and having a minimum (resp. maximum) at (−1,−1,−1) (resp. (1,1,1) ).
