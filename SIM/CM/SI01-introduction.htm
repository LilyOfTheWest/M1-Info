What is image synthesis?

  • Creates synthetic images from numerical models
  • Images are created to be seen by humans
  • Goal: Generates plausible/legible images for the human eyes

Simplified pipeline

  • Need a "model " of the world
    • 2D (images)
    • 3D (world coords)
    • 4D (+ time)
  • A simplified version of the model involves:
    • a camera
    • some 3D objects
    • some materials, appearance properties associated to (part of) the models
    • some lights (ambiant / directionnal / spot / etc)
    • some renderings (sampling strategies / antialiasing / post-processing / etc)

The 3D models

What are they?
  • Points / curves / surfaces / volumes, possibly animated
How are they represented?
  • Infinite possibilities! (how would you represent a simple square? a smooth curve?)
  • Surfaces are often reduced to quadrangular and/or triangular meshes (simple/GPUs)
    • This is what we will use during this course
    • Example:

  • Other possible representations:
    • using points (Particules / splats)
    • using boundary curves (B-rep)
    • using operations on simple primitives (CSG)
    • using images (billboards)
    • using implicit functions
    • ...
How are they animated?
  • Using simulated physic models (example for soft body locomotion)
  • Using keyframing / skeletons
  • Do the animations always need to be physically correct?

How are they created?
  • From scratch
    • by artists / designers
    • using modelisation tools: blender (free), 3DSMax, Maya, ZBrush, Modo, Meshlab (free), etc

  • From real data
    • using 3D scanners
    • using a set of images

The material / appearance

  • What is the material of the object? (plastic / marble / glass / etc)
  • What are its properties? (matte / glossy / mirror / transparent / translucent / etc)
  • Does its color/reflectance properties change spatially?


How can we represent all these effects?
  • Micro-effects (material): usually using a function called BRDF (Bidirectional Reflectance Distribution Function)

The BRDF defines how light is reflected on a surface
 Plastic material (Phong BRDF)
 Complex materials (translucency). Needs more dimensions (BSSRDF)
  • Meso-effects (details): mostly with textures (texture/bump/normal mapping)
Texture mapping
 normal mapping
  • Macro-effects (geometry): for instance with displacement maps. See a video here: ndo2

  • Hybrid representations: combines geometry, textures and/or volumes to represent natural scenes, medical data, etc
Clouds [Bouthors et al. 2008]

Lighting effects

  • No Light -> No images
  • Image synthesis consists in modelizing light behavior:
    • physical properties (intensity/color/etc)
    • directory (straight lines bouncing on surfaces/projections/reflections)
    • interaction with surfaces (material - BRDF/shadows/textures/reflectance)

The renderings

  • Goal: obatining a color for each pixel of the output image
    • What are the observed surfaces?
    • What are the associated materials/lights/textures?
  • Rasterization: fills the projected vector primitives (triangles) with colors
  • Raytracing: follows the inverse light trajectory

  • Sampling issue (aliasing): filtering / multisampling / supersampling (for anti-aliasing)


  • Storing colors:
    • Type: integer/float/etc?
    • format: number of colors / compression?
    • HDR images: tone mapping?
